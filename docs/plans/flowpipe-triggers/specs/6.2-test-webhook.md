# Task 6.2: Test Webhook Endpoint

## Context

Implement the webhook test endpoint to simulate webhook execution from the UI.

Reference: [plan.md](../plan.md) § 6.

## Steps

1. **Add `HandleTestWebhook(w, r)`**:
   - POST `/api/flows/webhooks/{triggerId}/test`
   - Accepts optional `X-Webhook-Secret` header
   - Accepts optional body in request

2. **Simulate webhook execution**:
   - Parse trigger config
   - Generate test payload
   - Call Flowpipe API to execute pipeline

3. **Return test results**:
   - Execution ID from Flowpipe
   - Timestamp
   - Test payload sent
   - Execution status link

4. **Add UI test button**:
   - In TriggerForm, "Test Webhook" button
   - Shows modal with test results
   - Includes request/response details

## Files

| Path | Action |
|------|--------|
| `api/internal/handler/flow.go` | Add test handler |

## Acceptance Criteria

- [ ] Test webhook endpoint works
- [ ] Test payload generated
- [ ] Test results returned
- [ ] UI button works

## Validation

```bash
cd api && go test ./handler -v -run TestTestWebhook
```
