# Task 1.4: Trigger Model Types

## Context

Expand the model layer to include all trigger-related types needed for API responses and HCL generation. This task adds yet-to-defined structs that will be used by the service layer.

Reference: [plan.md](../plan.md) § 3.1, § 5.1.

## Steps

1. **Update `TriggerConfig` struct** in `model/trigger.go`:
   - Add `WebhookConfig` struct:
     ```go
     type WebhookConfig struct {
         Description string `yaml:"description,omitempty"`
     }
     ```
   - Add `ScheduleConfig` struct:
     ```go
     type ScheduleConfig struct {
         Cron      string `yaml:"cron"`
         Timezone  string `yaml:"timezone,omitempty"`
     }
     ```
   - Add `AlertConfig` struct:
     ```go
     type AlertConfig struct {
         Source    string `yaml:"source"`
         Condition string `yaml:"condition"`
     }
     ```
   - Add `HTTPConfig` struct:
     ```go
     type HTTPConfig struct {
         URL    string `yaml:"url"`
         Method string `yaml:"method,omitempty"`
         Body   string `yaml:"body,omitempty"`
     }
     ```
   - Add `NotificationConfig` struct:
     ```go
     type NotificationConfig struct {
         Source  string `yaml:"source"`
         Channel string `yaml:"channel,omitempty"`
         Conditions []string `yaml:"conditions,omitempty"`
     }
     ```

2. **Add `TriggerState` struct**:
   - `Enabled bool` (default true)
   - `LastRun *time.Time`
   - `NextRun *time.Time`
   - `Status string` (ready, error, paused)

3. **Add API response structures**:
   - `TriggerListResponse` with `Triggers []TriggerState`
   - `TriggerWebhookURLResponse` with `URL string`
   - `TriggerTestResponse` with `ExecutedAt time.Time`, `Status string`

4. **Add flow trigger metadata**:
   - Add `Triggers []string` to `model.Flow` (list of trigger IDs attached to this flow)
