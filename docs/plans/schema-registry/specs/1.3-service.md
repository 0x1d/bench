# Task 1.3: Phase 1 — Service

## Context

Implement the schema service that provides `List()`, `Get()`, and `Content()`. Path resolution must reuse the same security rules as REST spec loading: resolve relative to config dir, reject `..` and path traversal.

Reference: [plan.md](../plan.md) § 5 Phase 1, § 8.  
Reference implementation: `api/internal/service/rest/service.go` — `Spec()` method for path resolution.

## Steps

1. **Create** `api/internal/service/schema/service.go`

2. **Define Service struct** and `NewService()` constructor

3. **Implement List()** `[]model.SchemaResource`:
   - Call `config.SchemaEntries()`
   - Map each to `SchemaResource` with `ID`, `Label`, `Type`, `Source: { Path: e.Source.Path }`
   - Return slice

4. **Implement Get(id string)** `(*model.SchemaResource, error)`:
   - Call `config.SchemaByID(id)`
   - If nil, return `ErrSchemaNotFound` (define sentinel error)
   - Map to `SchemaResource` and return

5. **Implement Content(id string)** `([]byte, error)`:
   - Call `Get(id)`; if error, return it
   - Get `config.ConfigDir()` for base path
   - Resolve `entry.Source.Path` relative to baseDir:
     - Use `filepath.Clean(entry.Source.Path)`
     - Reject if `clean` is `..` or starts with `..\` or `../`
     - Resolve `absPath := filepath.Join(baseDir, clean)`
     - Verify `filepath.Rel(baseDir, absPath)` does not start with `..`
   - Return `os.ReadFile(absPath)`

6. **Define errors**:
   - `ErrSchemaNotFound` — schema id not found

## Acceptance Criteria

- [ ] `List()` returns all configured schemas
- [ ] `Get("unknown")` returns `ErrSchemaNotFound`
- [ ] `Content("unknown")` returns `ErrSchemaNotFound`
- [ ] `Content(id)` returns file bytes when path is valid
- [ ] `Content(id)` returns error when path contains `..` (path traversal attempt)
- [ ] `Content(id)` returns error when file does not exist
