# Task 5.1: Messaging Config

## Context

Add `resources.messaging[]` for Kafka/MQTT brokers. Each entry references an AsyncAPI schema via schemaId. This is a design and config task; actual broker connection is future work.

Reference: [plan.md](../plan.md) § 5 Phase 5.

## Steps

1. **Design MessagingEntry** (config struct):
   - `ID string` — unique id
   - `Label string` — display name
   - `SchemaID string` — references resources.schemas[].id (must be asyncapi type)
   - `BrokerURL string` (optional) — e.g. Kafka bootstrap, MQTT broker
   - Additional fields TBD for Kafka/MQTT specifics

2. **Add to ResourcesConfig**:
   - `Messaging []MessagingEntry` (yaml: `messaging`)

3. **Add validation**:
   - Unique id
   - schemaId required and must reference schema with type `asyncapi`
   - SchemaByID(schemaId) must exist

4. **Add config helpers**:
   - `MessagingResources()` — returns configured messaging entries
   - `MessagingByID(id)` — returns entry by id

5. **Update config.example.yaml** with example messaging entry
