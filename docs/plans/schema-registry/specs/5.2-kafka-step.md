# Task 5.2: Phase 5 — Kafka Step

**Task ID**: 5.2  
**Phase**: 5 — AsyncAPI & Future Step Types  
**Prerequisites**: 5.1 (Messaging Config)

## Context

Add flow step type for publishing to Kafka (or generic event publish). Step config: schemaId, channel/topic, payload. Generate Flowpipe HCL.

Reference: [plan.md](../plan.md) § 5 Phase 5.  
Note: Flowpipe may not have native Kafka step; this spec may require custom step or HTTP-based publish. Adjust based on Flowpipe capabilities.

## Steps

1. **Define step type**:
   - Type string: `"kafka"` or `"publish"` (choose based on Flowpipe)
   - Config: `schemaId`, `channel` (or `topic`), `payload` (JSON string)

2. **Add to flow model** (if needed):
   - FlowStepConfigKafka or similar in model/flow.go

3. **Implement stepKafka in flow service**:
   - Generate Flowpipe HCL for the step
   - If Flowpipe has Kafka step: use it; else use HTTP step to Kafka REST proxy or similar
   - Document any assumptions (e.g. Kafka REST proxy, schema registry URL)

4. **Register step type** in generateHCL switch

5. **Update flow editor** step types:
   - Add "Kafka" or "Publish" to addable step types
