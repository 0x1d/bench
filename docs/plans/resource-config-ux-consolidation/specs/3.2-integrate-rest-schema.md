# Task 3.2: Integrate REST and Schema browser pages

## Context

REST resources may reference `schemaId` from `resources.schemas`. The REST settings form needs a **schema picker** populated from current config (or API) and optional link to `#schemas`. Schema registry editing belongs on the Schema browser page. See [plan.md § Per-page vs global](../plan.md#per-page-vs-global).

## Steps

1. **REST** ([`rest-page.tsx`](../../../../ui/src/pages/rest-page.tsx)): Add `Browse | Settings` (or list + Settings) tabs; move REST CRUD + auth into Settings; ensure `schemaId` select lists registered schemas.
2. **Schema browser** ([`schema-browser-page.tsx`](../../../../ui/src/pages/schema-browser-page.tsx)): Add work vs Settings tabs; Settings hosts schema registry add/edit/remove.
3. Keep `TabsList` compact (non-full-width).
4. Remove reliance on monolithic Configuration for these resource types once functional.
