# Task 3.3: Phase 3 — Schema Browser (Optional)

**Task ID**: 3.3  
**Phase**: 3 — Schema Registry UI & Standalone Schemas  
**Prerequisites**: 3.1, 3.2  
**Optional**: Yes

## Context

Schema browser page: like REST page but for any schema type. User selects a schema, sees parsed structure (operations, channels, etc.) and can explore.

Reference: [plan.md](../plan.md) § 5 Phase 3 Optional.

## Steps

1. **Create schema browser page** (or section):
   - Route/hash: e.g. `#schemas` or `#resources/schemas`
   - List schemas from `GET /api/schemas`
   - Select schema → fetch content → parse and display

2. **Display by type**:
   - OpenAPI: show operations (reuse RestClient or similar component)
   - AsyncAPI: show channels and operations
   - JSON Schema: show schema structure (properties, etc.)

3. **Add to app navigation**:
   - Link in sidebar or resources section
