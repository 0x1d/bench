# Task 1.4: Update UI types, form, and list components

## Context

The UI currently has both `webhook` and `http` in the `TriggerType` union and form. The webhook type only shows a `pipeline` field. The HTTP type incorrectly shows `url`, `method`, `body` fields. After the backend change, the `http` type should show `pipeline`, `args` (key-value pairs), and `execution_mode` (dropdown: asynchronous/synchronous). The "copy webhook URL" button should now appear for `http` type triggers instead of `webhook`.

## Steps

1. **Update `ui/src/services/api.ts`**:
   - Remove `'webhook'` from `TriggerType` union: `type TriggerType = 'schedule' | 'alert' | 'http' | 'notification'`
   - Update `TriggerConfig` interface: remove webhook-specific comments, ensure `args` field is typed as `Record<string, string> | undefined`, add `executionMode?: string`

2. **Update `ui/src/components/resource-config/trigger-form.tsx`**:
   - Remove `'webhook'` from `TRIGGER_TYPES` array
   - Remove `WebhookConfigFields` component entirely
   - Rewrite `HttpConfigFields` component to show:
     - Pipeline selector (same as before)
     - Args key-value pairs (like Schedule args — add/remove key-value rows)
     - Execution mode dropdown: `asynchronous` (default), `synchronous`
   - Update the conditional render: remove `draft.type === 'webhook'` block

3. **Update `ui/src/components/resource-config/trigger-list.tsx`**:
   - Remove `webhook` from `TRIGGER_TYPE_COLORS` record
   - Update the "copy webhook URL" button condition from `trigger.type === 'webhook'` to `trigger.type === 'http'`

4. **Update `ui/src/components/Flows/FlowTriggersList.tsx`**:
   - Remove `webhook` from any type colors/constants
   - Update webhook URL copy button condition from `=== 'webhook'` to `=== 'http'`
   - Update default draft type from `'webhook'` to `'http'`

5. **Update `ui/src/pages/triggers-page.tsx`**:
   - Update default draft type from `'webhook'` to `'http'`
