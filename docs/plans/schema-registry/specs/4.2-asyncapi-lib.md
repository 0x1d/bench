# Task 4.2: AsyncAPI Lib

## Context

Create AsyncAPI parser. Extract channels, operations (publish/subscribe), and message payload schemas. Structure similar to openapi.ts for OpenAPI.

Reference: [plan.md](../plan.md) § 3.1, § 5 Phase 4.  
AsyncAPI spec: https://asyncapi.com/docs/reference/specification/latest

## Steps

1. **Create** `ui/src/lib/asyncapi.ts`

2. **Define types**:
   - `AsyncAPIChannel` — channel name, publish/subscribe operations
   - `AsyncAPIOperation` — operationId, summary, message schema ref
   - `AsyncAPIParsed` — channels, operations list

3. **Implement parseAsyncAPI(content: string)**:
   - Parse JSON or YAML
   - Extract `channels` object
   - For each channel: extract `publish`/`subscribe` with `operationId`, `summary`, `message`
   - Resolve message payload schema (may use `schemaFormat`, `payload`)
   - Return structured data for UI consumption

4. **Implement getMessagePayloadSchema** (if needed):
   - Resolve $ref in AsyncAPI doc for message schemas
   - Return properties for form generation (similar to getRequestBodySchema in openapi.ts)

5. **Integrate with schema-registry.ts**:
   - parseSchema(content, 'asyncapi') calls parseAsyncAPI and returns `{ type: 'asyncapi', data }`
