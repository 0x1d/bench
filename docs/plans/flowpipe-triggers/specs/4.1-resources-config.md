# Task 4.1: Triggers Config Section

## Context

Add the triggers management UI to the Configuration page, allowing users to add, edit, and remove flowpipe triggers.

Reference: [plan.md](../plan.md) § 4.

## Steps

1. **Add triggers section to Configuration page**:
   - Find existing Configuration/Settings page
   - Add new "Flowpipe Triggers" section
   - Use same pattern as Schemas section

2. **Add trigger list component**:
   - Display triggers in table/grid
   - Columns: ID, Label, Flow, Type, Workspace, Status
   - Filter by type/workspace

3. **Add trigger add/edit dialog**:
   - Form with all fields from TriggerEntry
   - Type selector (webhook/schedule/alert/http/notification)
   - Dynamic form based on type
   - Flow list dropdown
   - Workspace dropdown

4. **Implement CRUD API calls**:
   - `GET /api/flows/triggers` on mount
   - `POST /api/flows/triggers` on create
   - `PUT /api/flows/triggers/{id}` on update
   - `DELETE /api/flows/triggers/{id}` on delete

5. **Add validation**:
   - Client-side validation for required fields
   - Show server errors in UI
