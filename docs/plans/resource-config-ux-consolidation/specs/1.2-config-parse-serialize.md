# Task 1.2: Extract config parse and serialize

## Context

[`configuration-page.tsx`](../../../../ui/src/pages/configuration-page.tsx) contains `parseConfigToState`, `stateToYaml`, `emptyState`, and shared TypeScript interfaces. Moving them to a dedicated module enables reuse by feature pages and hooks without duplicating YAML logic. See [plan.md § Architecture](../plan.md#architecture).

## Steps

1. Create `ui/src/lib/resource-config/` (or `ui/src/features/resource-config/` per team preference; stay consistent with [plan.md](../plan.md)).
2. Move types (`FilesystemResource`, `DatabaseResource`, `RestResource`, `SchemaResourceEntry`, `ResourceFormState`, `FlowsConfig`, `InfrastructureConfig`, `AgentConfig`, workspace types, auth types, etc.) into one or more `*.ts` files.
3. Move `emptyState`, `parseConfigToState`, and `stateToYaml` (or equivalent) with **no behavior change** — output YAML should match current serialization for the same state.
4. Re-export from the module; update `configuration-page.tsx` to import from the new module and delete inlined duplicates.
5. Add minimal unit tests for parse → serialize round-trip on a fixture string if the repo already uses Vitest for `ui/` (optional but recommended for regression).
