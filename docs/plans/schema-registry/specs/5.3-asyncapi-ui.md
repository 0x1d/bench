# Task 5.3: Phase 5 — AsyncAPI UI

## Context

Integrate AsyncAPI parser into flow step panel. When step uses AsyncAPI schema, show channel/topic picker and payload form.

Reference: [plan.md](../plan.md) § 5 Phase 5.

## Steps

1. **Flow step panel for Kafka/publish step**:
   - Add KafkaStepConfig (or PublishStepConfig) component
   - Similar structure to HttpStepConfig

2. **Schema selector**:
   - Dropdown: messaging resources (or schemas filtered by type asyncapi)
   - On select: fetch schema content via `GET /api/schemas/{id}/content`
   - Parse with parseAsyncAPI

3. **Channel/topic picker**:
   - Populate from parsed AsyncAPI channels
   - User selects channel
   - Store in step config

4. **Payload form**:
   - Use getMessagePayloadSchema (or equivalent) from asyncapi.ts
   - Generate form fields from message schema
   - Store JSON payload in step config

5. **Wire to FlowStepPanelContent**:
   - When step.type === 'kafka' (or 'publish'), render KafkaStepConfig

## Files

| Path | Action |
|------|--------|
| `ui/src/components/flow-step-panel-content.tsx` | Modify |
| `ui/src/services/api.ts` | Add fetchSchemaContent if needed |

## Acceptance Criteria

- [ ] Kafka step panel shows schema selector
- [ ] Channel picker populated from AsyncAPI
- [ ] Payload form generated from message schema
- [ ] Config saved correctly to flow
