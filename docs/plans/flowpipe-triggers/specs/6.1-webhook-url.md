# Task 6.1: Webhook URL Generation

## Context

Implement webhook URL generation for trigger testing, following Flowpipe's webhook pattern.

Reference: [plan.md](../plan.md) § 6.

## Steps

1. **Add `GenerateWebhookURL(trigger *TriggerEntry)`**:
   - Format: `{FlowpipeURL}/api/v0/webhook/{triggerId}`
   - For local dev, may need to proxy through Bench API
   - Include workspace in URL or header

2. **Add webhook proxy endpoint**:
   - POST `/api/flows/webhooks/{triggerId}/proxy`
   - Receive webhook request
   - Parse body
   - Call Flowpipe API to trigger execution

3. **Store webhook state**:
   - Track webhook triggers
   - Record last webhook received time
   - Store payload checksum for deduplication

4. **Handle webhook auth**:
   - Flowpipe handles webhook signature verification
   - Bench may add additional auth if needed
   - Consider allowing secret in config
