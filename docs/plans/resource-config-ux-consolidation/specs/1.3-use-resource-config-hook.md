# Task 1.3: useResourceConfig hook

## Context

`saveConfig` replaces the entire YAML file. Any feature page that edits one slice must **read** current config, **merge** the slice, **write** back. Centralize this in `useResourceConfig` (or similarly named hook) with TanStack Query: shared query key for raw config, `saveSlice` that parses, applies merge, serializes, calls `saveConfig`, then invalidates `useStatus` and relevant feature queries — mirroring current `persistState` in the monolith. See [plan.md § Risk](../plan.md#security).

## Steps

1. Implement `useResourceConfig` in `ui/src/lib/resource-config/` (or adjacent file) using `fetchConfig` / `saveConfig` from [`api.ts`](../../../../ui/src/services/api.ts).
2. Expose: loading/error state, parsed `ResourceFormState` (or accessors), and `updateSlice` / `savePartial` API that **never** drops unrelated top-level keys when merging.
3. Wire `configuration-page.tsx` to use the hook for load + save paths that previously used local state + `persistState` (incremental migration acceptable if behavior is identical).
4. Document merge semantics in a short comment (read-modify-write).
