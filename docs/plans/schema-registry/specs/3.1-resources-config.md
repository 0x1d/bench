# Task 3.1: Phase 3 — Resources Config UI

**Task ID**: 3.1  
**Phase**: 3 — Schema Registry UI & Standalone Schemas  
**Prerequisites**: Phase 2 complete

## Context

Add schemas section to the Resources config page. Users can list, add, edit, and remove schema entries. Schemas are persisted to config.yaml on save.

Reference: [plan.md](../plan.md) § 5 Phase 3.  
Reference implementation: `ui/src/pages/resources-config-page.tsx` — databases and rest sections.

## Steps

1. **Add schemas to parsed config state**:
   - Extend `ResourcesState` (or equivalent) with `schemas: SchemaResource[]`
   - Parse `resources.schemas` from YAML in `parseConfigFromYaml`
   - Map to `{ id, label, type, source: { path } }`

2. **Add schema list UI**:
   - New section "Schemas" (similar to REST or Databases)
   - Table or card list: id, label, type, source path
   - "Add schema" button
   - Edit and Remove actions per row

3. **Add schema form panel**:
   - Fields: id (required), label, type (dropdown: openapi, asyncapi, json-schema), source path (required)
   - Save / Cancel buttons
   - Validation: id and path required; id unique

4. **Persist on save**:
   - Include `schemas` in the YAML output when saving config
   - Emit under `resources.schemas` with correct structure

5. **Remove schema**:
   - Confirmation dialog before delete
   - Remove from state and persist
