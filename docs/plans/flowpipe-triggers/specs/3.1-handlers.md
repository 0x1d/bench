# Task 3.1: Trigger API Handlers

## Context

Implement HTTP handlers for all trigger CRUD operations, following the same pattern as existing handler files.

Reference: [plan.md](../plan.md) § 3.3.

## Steps

1. **Add to `api/internal/handler/flow.go`**:
   - `HandleTriggersList(w, r)` - GET `/api/flows/triggers`
   - `HandleFlowTriggersList(w, r)` - GET `/api/flows/{flowId}/triggers`
   - `HandleTriggerGet(w, r)` - GET `/api/flows/triggers/{triggerId}`
   - `HandleTriggerCreate(w, r)` - POST `/api/flows/triggers`
   - `HandleTriggerUpdate(w, r)` - PUT `/api/flows/triggers/{triggerId}`
   - `HandleTriggerDelete(w, r)` - DELETE `/api/flows/triggers/{triggerId}`

2. **Implement request bodies**:
   - Create: `POST` body is `TriggerEntry`
   - Update: `PUT` body is `TriggerEntry`

3. **Implement responses**:
   - List: `{ triggers: TriggerState[] }`
   - Get: `TriggerState`
   - Create: `TriggerState` with status 201
   - Delete: status 204

4. **Add error handling**:
   - 404 for not found
   - 400 for invalid input
   - 500 for server errors
   - 409 for conflicts

5. **Add workspace query param**:
   - Filter triggers by workspace
   - Default to "default" workspace
