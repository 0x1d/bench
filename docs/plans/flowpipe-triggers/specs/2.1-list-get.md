# Task 2.1: List and Get Triggers

## Context

Implement `ListTriggers()` and `GetTrigger()` methods to read trigger definitions from Flowpipe mod files (.fp files).

Reference: [plan.md](../plan.md) § 3.2.

## Steps

1. **Add `ListTriggers()` to `Service`**:
   - Read flows directory (config.FlowsPath())
   - Walk all `.fp` files
   - Parse each file for trigger blocks using HCL/regex
   - Return `[]TriggerState`

2. **Add `GetTrigger(flowID, triggerID string)`**:
   - Find the flow's `.fp` file
   - Parse handle trigger block with given ID
   - Return `*TriggerState`

3. **Add `GetTriggerFromConfig(flowID, triggerID string)`**:
   - Get trigger from config.yaml
   - Cross-reference with flow file

4. **Add `flow.fp` file parser**:
   - Parse HCL trigger blocks
   - Extract: `pipeline`, `cron`, `source`, `condition`, etc.
   - Handle both inline and module-based flows

5. **List triggers by workspace**:
   - Parse `workspaces.fpc` for workspace definitions
   - Return triggers filtered by workspace
