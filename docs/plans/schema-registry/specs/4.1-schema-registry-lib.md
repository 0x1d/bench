# Task 4.1: Schema Registry Lib

## Context

Create a generic schema parsing layer. `detectSchemaType` infers type from content. `parseSchema` returns a union of parsed structures. Keeps `openapi.ts` for OpenAPI-specific helpers.

Reference: [plan.md](../plan.md) § 5 Phase 4.

## Steps

1. **Create** `ui/src/lib/schema-registry.ts`

2. **Implement detectSchemaType(content: string)**:
   - Parse content as JSON or YAML (reuse pattern from openapi.ts)
   - Check root keys: `openapi` → `'openapi'`, `asyncapi` → `'asyncapi'`
   - Check `$schema` for JSON Schema (e.g. `http://json-schema.org/...`) → `'json-schema'`
   - Return `'openapi' | 'asyncapi' | 'json-schema' | 'unknown'`

3. **Define SchemaParsed union type**:
   - `{ type: 'openapi'; data: OpenAPIParsed }`
   - `{ type: 'asyncapi'; data: AsyncAPIParsed }` (stub for 4.2)
   - `{ type: 'json-schema'; data: JsonSchemaParsed }` (minimal stub)

4. **Implement parseSchema(content: string, type: string)**:
   - If type `openapi`: use existing `parseOpenAPIOperationsGrouped` or similar from openapi.ts; return `{ type: 'openapi', data }`
   - If type `asyncapi`: delegate to asyncapi.ts (4.2) or return stub
   - If type `json-schema`: parse and return minimal structure
   - If `unknown`: return `{ type: 'unknown', data: null }`

5. **Re-export** OpenAPI helpers from openapi.ts or import as needed; avoid duplicating logic.
