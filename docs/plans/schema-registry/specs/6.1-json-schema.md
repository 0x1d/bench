# Task 6.1: JSON Schema

## Context

Support `type: json-schema` in schema registry. Config and API already support it (Phase 1). This task adds validation and optional use cases.

Reference: [plan.md](../plan.md) § 5 Phase 6.

## Steps

1. **Verify schema registry**:
   - `type: json-schema` already accepted in config (Phase 1)
   - Schema service Content() returns schema content
   - No code changes if already working

2. **Optional: validate schema structure**:
   - On Content() or separate validation: parse JSON and validate it's valid JSON Schema (has `$schema` or valid structure)
   - Return validation error if malformed

3. **Optional: POST /api/schemas/validate**:
   - Request: `{ schemaId, payload }`
   - Validate payload against schema using JSON Schema validator
   - Response: `{ valid: bool, errors?: [...] }`
   - Requires Go JSON Schema library (e.g. github.com/xeipuuv/gojsonschema)

4. **Use cases (design only)**:
   - Transform step: validate output against schema — document in plan
   - Config validation: validate config.yaml — document in plan
