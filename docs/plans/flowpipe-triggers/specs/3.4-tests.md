# Task 3.4: API Integration Tests

## Context

Write integration tests that test the full HTTP request/response cycle for trigger endpoints.

Reference: [plan.md](../plan.md) § 9.

## Steps

1. **Create test server**:
   - Use `httptest.NewServer()` with router from `routes.go`
   - Set up test database/config

2. **Write tests for each endpoint**:
   - `TestListTriggersAPI()`: Create triggers, list them
   - `TestCreateTriggerAPI()`: POST valid/invalid triggers
   - `TestUpdateTriggerAPI()`: PUT to update trigger
   - `TestDeleteTriggerAPI()`: DELETE triggers
   - `TestTestTriggerAPI()`: POST to test endpoint
   - `TestWebhookURIAPI()`: GET webhook URL

3. **Test request bodies**:
   - Valid JSON triggers accepted
   - Invalid JSON rejected
   - Missing required fields rejected

4. **Test responses**:
   - Correct status codes
   - Correct response format
   - Error responses include message

5. **Test authentication**:
   - Requests without token rejected
   - Requests with invalid token rejected
   - Requests with valid token accepted
