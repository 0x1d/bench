# Task 1.2: Trigger Config Helpers

## Context

Add helper functions to load and parse triggers from config.yaml, following the same pattern as `SchemaEntries()` and `Databases()`.

Reference: [plan.md](../plan.md) § 5.1.

## Steps

1. **Add `FlowpipeTriggersConfig` struct** to `config.go`:
   - Put at top level with `FlowsConfig`, `InfrastructureConfig`, `AgentConfig`
   - Contains `Triggers []TriggerEntry`

2. **Update `Config` struct**:
   - Add `FlowpipeTriggers *FlowpipeTriggersConfig` field

3. **Add `TriggerEntries()` function** (same pattern as `SchemaEntries()`):
   - Returns `[]TriggerEntry` from `cfg.FlowpipeTriggers.Triggers`
   - Skip entries with empty `ID` or `Flow`
   - Default `Label` to `ID` when empty
   - Return empty slice on error

4. **Add `TriggerEntriesWithError()` function**:
   - Same as `TriggerEntries()` but preserves load errors

5. **Add `TriggerByID(id string)` function**:
   - Returns `*TriggerEntry` or nil
   - Iterate `TriggerEntries()` and match by `ID`

6. **Add `WorkspaceTriggers(workspace string)` function**:
   - Returns triggers for a specific workspace
   - Filter by `Workspace` field, default to "default"

7. **Update `config.example.yaml`**:
   - Add `flowpipe_triggers` section:
     ```yaml
     flowpipe_triggers:
       - id: daily-report-schedule
         label: Daily Report
         flow: daily_report
         type: schedule
         config:
           description: "Run daily at 9 AM"
           pipeline: pipeline.daily_report
           schedule:
             cron: "0 9 * * *"
             timezone: "UTC"
       - id: webhook-trigger
         label: Webhook Trigger
         flow: data_sync
         type: webhook
         config:
           description: "Trigger on webhook"
           pipeline: pipeline.data_sync
     ```
