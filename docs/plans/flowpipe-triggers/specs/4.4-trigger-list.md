# Task 4.4: Trigger List Component

## Context

Create a standalone TriggerList component with filtering and status management.

Reference: [plan.md](../plan.md) § 4.

## Steps

1. **Create `ui/src/components/Flows/TriggerList.tsx`**:
   - Reusable list component
   - Can be used on Triggers page or in flow panel

2. **Add filters**:
   - Type dropdown
   - Workspace dropdown
   - Flow search
   - Combined filters work

3. **Add actions column**:
   - Edit button
   - Delete button (with confirmation)
   - Test button (tooltip: "Simulate trigger execution")

4. **Add status indicators**:
   - Ready (green)
   - Error (red)
   - Paused (gray)

5. **Add pagination** (optional for large lists):
   - Page size selector
   - Next/prev buttons

6. **Add empty state**:
   - "No triggers configured"
   - CTA to add trigger
