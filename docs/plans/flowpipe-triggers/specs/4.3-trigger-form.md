# Task 4.3: Trigger Form Dialog

## Context

Implement the create/edit dialog with type-specific form fields for triggers.

Reference: [plan.md](../plan.md) § 4.

## Steps

1. **Create `ui/src/components/Flows/TriggerForm.tsx`**:
   - Formik-based form or React Hook Form
   - Multi-step or tabbed interface
   - Pre-filled with existing data for edit mode

2. **Add type selector**:
   - Dropdown with types: webhook, schedule, alert, http, notification
   - Shows/hides fields based on type

3. **Add dynamic field groups**:
   - **webhook**: description, pipeline, (optional webhook config)
   - **schedule**: description, pipeline, cron, timezone
   - **alert**: description, pipeline, source, condition
   - **http**: description, pipeline, url, method, body
   - **notification**: description, pipeline, source, channel, conditions

4. **Add field validation**:
   - Required fields marked with *
   - Cron format validation
   - Pipeline reference validation (matches existing pipelines)

5. **Add submit handler**:
   - Create: POST to `/api/flows/triggers`
   - Update: PUT to `/api/flows/triggers/{id}`
   - Close dialog on success
   - Show validation errors
