# Task 3.2: Test Trigger Handler

## Context

Implement the test trigger handler and webhook URL handler for simulating trigger execution.

Reference: [plan.md](../plan.md) § 6.

## Steps

1. **Add `HandleTriggerTest(w, r)`**:
   - POST `/api/flows/triggers/{triggerId}/test`
   - Body may contain payload: `{ payload: { ... } }`
   - Calls `TestTrigger(flowID, triggerID, payload)`
   - Returns test execution result

2. **Add `HandleTriggerWebhookURL(w, r)`**:
   - GET `/api/flows/triggers/{triggerId}/webhook`
   - Returns webhook URL for testing
   - URL includes workspace and trigger ID

3. **Generate webhook URL**:
   - Format: `{FlowpipeURL}/api/v0/webhook/{triggerId}`
   - Or generate local proxy URL if Flowpipe not available
   - Include workspace in URL or header

4. **Test handler with mock**:
   - Verify test payload format
   - Verify URL generation
