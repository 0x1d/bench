# Task 1.4: Handler & Routes

## Context

Add HTTP handlers for schema registry API. Register routes under `/api/schemas`. Follow patterns from `api/internal/handler/rest.go`.

Reference: [plan.md](../plan.md) § 4.3.

## Steps

1. **Create** `api/internal/handler/schema.go`

2. **HandleSchemaList**:
   - Method: GET only
   - Call `schemaSvc.List()`
   - Response: `{ "schemas": [...] }` — JSON array of `SchemaResource`
   - Content-Type: application/json
   - Status: 200

3. **HandleSchemaGet**:
   - Method: GET only
   - Path param: `id` (from `r.PathValue("id")`)
   - Trim space; if empty, 400 Bad Request
   - Call `schemaSvc.Get(id)`
   - If `ErrSchemaNotFound`, 404
   - Response: single `SchemaResource` as JSON
   - Status: 200

4. **HandleSchemaContent**:
   - Method: GET only
   - Path param: `id`
   - Trim space; if empty, 400
   - Call `schemaSvc.Content(id)`
   - If `ErrSchemaNotFound`, 404
   - Response: raw bytes (Content-Type: application/json or application/x-yaml based on content — same logic as `HandleRestSpec`)
   - Status: 200

5. **Register routes** in `api/internal/handler/routes.go`:
   - `GET /api/schemas` → HandleSchemaList
   - `GET /api/schemas/{id}` → HandleSchemaGet
   - `GET /api/schemas/{id}/content` → HandleSchemaContent
   - Order: more specific paths first (content before {id})

6. **Create schema service instance** (e.g. `var schemaSvc = schema.NewService()`)

## Files

## Acceptance Criteria

- [ ] `GET /api/schemas` returns 200 with schemas array
- [ ] `GET /api/schemas/{id}` returns 200 with schema object when id exists
- [ ] `GET /api/schemas/{id}` returns 404 when id not found
- [ ] `GET /api/schemas/{id}/content` returns raw spec content
- [ ] Content-Type for content: JSON or YAML based on first character
- [ ] POST to these endpoints returns 405 Method Not Allowed
