# Task 3.3: Register Trigger Routes

## Context

Register the new trigger API routes in the router, following the pattern in `routes.go`.

Reference: [plan.md](../plan.md) § 3.3.

## Steps

1. **Open `api/internal/handler/routes.go`**:
   - Add import for `github.com/0x1d/bench/api/internal/handler`

2. **Add trigger routes**:
   ```go
   r.GET("/flows/triggers", h.HandleTriggersList)
   r.GET("/flows/{flowId}/triggers", h.HandleFlowTriggersList)
   r.GET("/flows/triggers/{triggerId}", h.HandleTriggerGet)
   r.POST("/flows/triggers", h.HandleTriggerCreate)
   r.PUT("/flows/triggers/{triggerId}", h.HandleTriggerUpdate)
   r.DELETE("/flows/triggers/{triggerId}", h.HandleTriggerDelete)
   r.POST("/flows/triggers/{triggerId}/test", h.HandleTriggerTest)
   r.GET("/flows/triggers/{triggerId}/webhook", h.HandleTriggerWebhookURL)
   ```

3. **Add group prefix** (optional):
   - All routes under `/api/flows/triggers*` prefix
   - Keep existing flow routes consistent

4. **Verify routes work**:
   - Run API server
   - Test each endpoint with curl
   - Check 404 for non-existent trigger
