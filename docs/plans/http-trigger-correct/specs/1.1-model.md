# Task 1.1: Remove webhook type, correct HTTP model

## Context

Flowpipe only has `trigger "http"` — there is no `trigger "webhook"`. The current codebase has both `TriggerTypeWebhook` and `TriggerTypeHTTP` constants, with `WebhookConfig` and `HTTPConfig` structs. The `WebhookConfig` is actually the correct model for Flowpipe's HTTP trigger (it receives inbound requests). The `HTTPConfig` is wrong — it models an outbound HTTP callback which doesn't exist in Flowpipe.

This task removes `TriggerTypeWebhook` and `WebhookConfig`, and rewrites `HTTPConfig` to match the Flowpipe HTTP trigger spec: `pipeline`, `args` (map), and `execution_mode`.

## Steps

1. **Update `api/internal/model/trigger.go`**:
   - Remove `TriggerTypeWebhook` constant and `WebhookConfig` struct
   - Rewrite `HTTPConfig` to: `{Description, Pipeline, Args map[string]string, ExecutionMode string}` — remove `URL`, `Method`, `Body` fields
   - Remove `HTTP` field from `TriggerConfig` and rename the concept — the `http` type now uses `HTTPConfig` with the new fields
   - Update `MarshalJSON` to flatten `args`, `execution_mode` for HTTP type instead of `url`, `method`, `body`
   - Remove webhook-specific flat key logic from `MarshalJSON`

2. **Update `api/internal/config/config.go`**:
   - Remove `TriggerTypeWebhook` constant and `WebhookConfig` struct (duplicate definitions)
   - Rewrite `HTTPConfig` to match the model
   - Update `TriggerConfig` to remove `Webhook` pointer field
   - Update valid trigger types map to remove `webhook`

3. **Update `config.example.yaml`**: Change any `type: webhook` examples to `type: http`
