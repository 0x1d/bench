# Task 2.6: Service Unit Tests

## Context

Write comprehensive unit and integration tests for the trigger service layer.

Reference: [plan.md](../plan.md) § 9.

## Steps

1. **Create test fixtures**:
   - Sample `.fp` files with trigger blocks
   - Sample trigger configs
   - Mock data

2. **Write tests for each method**:
   - `TestListTriggers()`: Verify triggers parsed correctly
   - `TestGetTrigger()`: Verify specific trigger returned
   - `TestCreateTrigger()`: Verify trigger added to file
   - `TestUpdateTrigger()`: Verify trigger updated
   - `TestDeleteTrigger()`: Verify trigger removed
   - `TestTestTrigger()`: Verify test execution

3. **Write integration tests**:
   - Full CRUD lifecycle test
   - Multiple triggers in one file
   - Triggers across multiple flows

4. **Write validation tests**:
   - Invalid trigger types rejected
   - Duplicate IDs rejected
   - Missing required fields rejected

5. **Write error handling tests**:
   - File not found
   - Parse errors
   - Flowpipe connection errors
