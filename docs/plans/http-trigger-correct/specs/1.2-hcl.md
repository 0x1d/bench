# Task 1.2: Correct HCL generation and parsing for HTTP trigger

## Context

The HCL generation in `BuildTriggerHCLBlock` currently generates `url`, `method`, `body` for HTTP triggers, which is incorrect. Flowpipe's HTTP trigger uses `pipeline`, `args`, and `execution_mode` at the top level, with optional `method "post"` / `method "get"` blocks. The HCL parser also needs to be updated to read `args` blocks and `execution_mode` from HTTP trigger blocks, and stop parsing `url`/`method`/`body`.

## Steps

1. **Update `BuildTriggerHCLBlock` in `api/internal/service/flow/trigger.go`**:
   - Remove the `case model.TriggerTypeHTTP` block that outputs `url`, `method`, `body`
   - Replace with correct HTTP trigger output:
     ```
     args = {
       key = "value"
     }
     execution_mode = "asynchronous"  // if set
     ```
   - Remove `case model.TriggerTypeWebhook` (no-op case)

2. **Update `parseTriggerBlockContent` in `api/internal/service/flow/trigger.go`**:
   - Remove `case "webhook"` block
   - Update `case "http"` to parse `args` and `execution_mode` instead of `url`/`method`/`body`
   - Use existing `parseHCLArgs` function for args extraction

3. **Update `triggerEntryToModelConfig` and `configTriggerEntryFromModel`**:
   - Remove webhook case handling
   - Update HTTP case to use new fields
