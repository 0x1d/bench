# Task 3.2: REST Form (schemaId)

## Context

Add schemaId dropdown to REST resource form. When user selects a schema (openapi type), REST resource uses it. Keep openapiSpec as fallback when schemaId is empty.

Reference: [plan.md](../plan.md) § 5 Phase 3.

## Steps

1. **Add schemaId to REST form state**:
   - RestResource type includes `schemaId?: string`
   - Form state for add/edit REST includes schemaId

2. **Add schemaId dropdown** to REST resource form:
   - Label: "Schema" or "OpenAPI schema (from registry)"
   - Options: schemas filtered by `type === 'openapi'`
   - Include empty option: "None (use path below)"
   - On change: set schemaId; clear openapiSpec if desired, or keep both (backend uses schemaId when set)

3. **openapiSpec field**:
   - Show when schemaId is empty or "none"
   - Help text: "Path to spec file, or select a schema from registry above"
   - When schemaId selected, openapiSpec can be hidden or shown as fallback (optional)

4. **Help text**:
   - "Use a registered schema (recommended) or specify a file path. Schema takes precedence."

5. **Persist schemaId** when saving REST entry to config

## Acceptance Criteria

- [ ] Dropdown lists only openapi-type schemas
