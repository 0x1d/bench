# Task 2.1: Extract resource editor components

## Context

[`configuration-page.tsx`](../../../../ui/src/pages/configuration-page.tsx) contains large JSX blocks for add/edit flows for filesystem, database, REST, schema, flows path + workspaces, infrastructure, and agent. Extract **presentational** components that take draft state and callbacks so feature pages and the slim Configuration screen can reuse them. See [plan.md § Implementation Phases](../plan.md#implementation-phases).

## Steps

1. Identify each panel mode / form section in the current page (list + side panel pattern).
2. Extract components such as `FilesystemResourceSettings`, `DatabaseResourceSettings`, `RestResourceSettings` (+ auth subform), `SchemaResourceSettings`, `FlowsAndWorkspacesSettings`, `InfrastructurePathSettings`, `AgentSettingsForm` — names may vary; keep one component per concern.
3. Preserve validation messages and duplicate-id checks; centralize shared helpers if duplicated.
4. Replace inline JSX in `configuration-page.tsx` with these components (still on the monolith until Phase 3–4) so the page shrinks and compiles.
